# Plan before you code (earns treats 🦴)

For anything beyond a one-line change, take 20 seconds to plan before editing:

1. **Restate the goal** in one sentence — what should be true when you're done?
2. **List the files/functions** you expect to touch.
3. **Name the approach** and one alternative you're rejecting (and why).
4. **Call out risks** — anything that could break, or that you're unsure about.
5. **Then implement.**

Keep the plan short (a few bullet points), not a document. A good plan surfaces
misunderstandings before they become wasted work — and that's exactly the kind of
careful work that earns treats here.
